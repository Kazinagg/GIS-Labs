## 🗺 Лабораторная работа №1: Построение карт в изолиниях в программе Surfer

### 🎯 Цель работы
Построить карты в изолиниях в программе Golden Software Surfer на основе синтезированных данных.

### 📋 Общая постановка задачи
- Установить программу Golden Software Surfer.
- Использовать CSV-файл с 3-мя колонками данных (X: Easting, Y: Northing, Z: Elevation).
- Создать регулярный грид и выполнить интерполяцию значений высотных отметок.
- Построить и визуализировать топокарту в изолиниях.

### 🚀 Ход выполнения работы

**1. Исходные данные и программное обеспечение**  
Метод получения данных — синтез значений высотных отметок на основе комплексной математической функции, имитирующей природный ландшафт (фрактальный шум, горный массив, каньон).

*Функция распределения высот `Z = f(X, Y)`:*
> `100*exp(-((X-50)^2+(Y-50)^2)/1000) - 30*exp(-((X-Y)^2)/150) + 15*sin(X/6.5)*cos(Y/8.2) + 6*sin(X/2.3+1.5)*cos(Y/2.9-1) + 2*sin(X/0.8)*cos(Y/0.9) + 0.4*Y + 0.2*X`

*Параметры координатной сетки:* X и Y от 0 до 100, Шаг сетки (Increment): `0.25`.

**2. Генерация регулярного грида (сетки высот)**  
С помощью инструмента `Grid -> Function` задана составная формула рельефа. Программа сформировала матрицу высот (grid-файл) с плотной сеткой интерполяции.

**3. Построение двумерной карты (Contour Map) и 3D модели (3D Surface)**  
С помощью инструмента `Home -> New Map -> Contour` сгенерирована топографическая карта, настроен шаг сечения. Добавлена 3D-модель (3D Surface), демонстрирующая возвышенность, эрозионный врез и холмы. 

**4. Компоновка макета (Layout)**  
Объекты размещены на одном печатном листе. Настроены координатные оси и цветовая палитра.

## 🌐 Лабораторная работа №3: Создание модели глобуса с географической сеткой и триангуляцией

### 🎯 Цель работы
Разработать базовое ГИС-приложение для визуализации 3D-модели Земли, реализовать загрузку тематических глобальных карт и наложить прозрачную сетку координат.

### 🛠 Стек технологий
- **Среда разработки:** RAD Studio C++Builder.
- **Графический движок:** GLScene (VCL).
- **Интерфейс:** Компоненты VCL (`TTreeView`, `TImageList`, `TPanel`).

### ⚙️ Архитектура и техническая реализация
1. **3D-сцена:** Центр сцены — `TGLDummyCube` (ось вращения). Модель Земли — `TGLSphere`. Камера сфокусирована на центре куба.
2. **Управление слоями:** Интерфейс выбора реализован через `TTreeView`. При смене узла динамически обновляется текстура (`topography.jpg`, `temperature.jpg`).
3. **Географическая сетка:** Сетка (`unigrid.bmp`) наложена поверх Земли с помощью дополнительной сферы с увеличенным радиусом (`1.02`). Прозрачность реализована через `bmTransparency` и `tiaTopLeftPointColorTransparent`.
4. **Интерактивность:** Свободное вращение (Drag & Drop) через обработку `OnMouseDown` и `OnMouseMove`.

### 🎮 Управление в приложении
- **Зажатие ЛКМ по глобусу + движение** — вращение Земли.
- **Меню слева** — переключение между слоями.

## 📍 Лабораторная работа №4: Работа с GPS и картами при разработке под Android

### 🎯 Цель работы
Разработать мобильное приложение для навигации и определения местоположения пользователя с использованием ОС Android и среды RAD Studio (Delphi/FireMonkey).

### 🛠 Технические особенности реализации
1. **Runtime Permissions:** Реализован динамический запрос прав `ACCESS_FINE_LOCATION` и `ACCESS_COARSE_LOCATION` для Android 6.0+.
2. **Картографический сервис:** Использование платформы `OpenStreetMap` для обхода ограничений стандартных интентов Google Maps.
3. **Потокобезопасность:** Обработка ответа от `TGeocoder` в фоновом потоке. Вывод данных в UI обернут в `TThread.Queue` для предотвращения зависаний.

### 🧱 Используемые компоненты (FMX)
* `TLocationSensor` — получение географических координат.
* `TWebBrowser` — отображение веб-интерфейса карты.
* `TLabel` — вывод числовых значений и декодированного адреса.
* `TGeocoder` — обратное геокодирование.

### 💻 Основной листинг кода
Обработчик события изменения геопозиции (`OnLocationChanged`):

```pascal
procedure TForm1.LocationSensor1LocationChanged(Sender: TObject; const OldLocation, NewLocation: TLocationCoord2D);
var
  Lats, Longs, HTMLString: string;
  FS: TFormatSettings;
begin
  // Форматирование координат (разделитель - точка, обязательно для API)
  FS := TFormatSettings.Create('en-US');
  Lats := FloatToStr(NewLocation.Latitude, FS);
  Longs := FloatToStr(NewLocation.Longitude, FS);

  // Вывод координат на экран
  Label1.Text := Format('Широта: %s Долгота: %s', [Lats, Longs]);

  // Загружаем Карту
  URL := Format('https://www.openstreetmap.org/?mlat=%s&mlon=%s#map=16/%s/%s', [Lats, Longs, Lats, Longs]);
  WebBrowser1.Navigate(URL);

  // Обратное геокодирование (Координаты -> Адрес)
  if not Assigned(FGeocoder) then
  begin
    if Assigned(TGeocoder.Current) then
    begin
      FGeocoder := TGeocoder.Current.Create;
      FGeocoder.OnGeocodeReverse := OnGeocodeReverseEvent;
    end;
  end;

  if Assigned(FGeocoder) and not FGeocoder.Geocoding then
    FGeocoder.GeocodeReverse(NewLocation);
end;
```
